# Руководство по JSON API v1

Все маршруты ниже имеют общий префикс **`/api/v1`**. Полный путь на вашем сервере выглядит как `https://ваш-домен/api/v1/...`.

Интерактивная схема (Swagger UI) доступна по **`/docs`** только если **`APP_ENV`** не `production` (в проде отключена).

---

## Настройка доступа

1. В переменных окружения или в файле `.env` задайте **`API_TOKEN`** — длинную случайную строку (как пароль приложения).
2. Каждый запрос к API должен содержать **один** из заголовков:
   - **`Authorization: Bearer <ваш_токен>`**  
   - **`X-Api-Key: <ваш_токен>`**

Если **`API_TOKEN`** не задан или пустой, все эндпоинты API отвечают **`503 Service Unavailable`** с телом JSON:

```json
{"detail": "API token is not configured"}
```

При неверном или отсутствующем токене (при включённом **`API_TOKEN`**) ответ — **`401 Unauthorized`**:

```json
{"detail": "Invalid or missing API token"}
```

Такой запрос засчитывается как неудачная попытка. После **трёх** попыток с одного IP адрес блокируется на **24 часа**: дальнейшие запросы к **`/api/v1/*`** получают **`429 Too Many Requests`**, в теле JSON — сообщение о блокировке, в заголовке **`Retry-After`** — оставшиеся секунды до конца бана. Успешный запрос с верным токеном обнуляет счётчик неудачных попыток API для этого IP.

Дополнительно действует **лимит частоты** всех запросов к API с одного IP (по умолчанию **300 в минуту**, переменная **`API_RATE_LIMIT_PER_MINUTE`**, `0` = без лимита). При превышении — **`429`** с `{"detail": "Too many API requests"}`.

API **не использует CSRF** (в отличие от HTML-админки): достаточно токена в заголовке. Целевые URL при создании ссылок проверяются так же, как в админке (запрет CRLF, приватных адресов).

Тело ответов об ошибках — JSON с полем **`detail`** (строка или список при ошибках валидации).

---

## Формат данных

- Запросы с телом: **`Content-Type: application/json`**.
- Успешные ответы: JSON с кодами **`200`**, **`201`**. Удаление без тела — **`204 No Content`**.
- Идентификаторы ссылок и кликов — UUID в строковом виде в JSON и в пути URL.

---

## Проверка токена и окружения

### `GET /api/v1/me`

Возвращает служебную информацию (без обращения к БД по смыслу бизнес-логики).

**Пример ответа `200`:**

```json
{
  "api": "v1",
  "geoip_db_present": true
}
```

- **`geoip_db_present`** — загружен ли хотя бы один MMDB (город или страна) для геолокации кликов.

**Пример с `curl`:**

```bash
curl -sS -H "Authorization: Bearer $API_TOKEN" "https://example.com/api/v1/me"
```

## Ссылки

### `GET /api/v1/links`

Список ссылок, от новых к старым.

**Query-параметры:**

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `limit`  | `100`        | Размер страницы, от 1 до 500 |
| `offset` | `0`          | Смещение для пагинации |

**Ответ `200`:** объект с полями **`items`** (массив ссылок) и **`total`** (общее число ссылок в базе, без учёта `limit`/`offset`).

Элемент **`items[]`** содержит:

| Поле | Описание |
|------|----------|
| `id` | UUID ссылки |
| `slug` | Короткий код для пути редиректа `/r/{slug}` |
| `destination_url` | Целевой URL |
| `label` | Аккаунт (например URL TikTok) или `null` |
| `created_at`, `updated_at` | ISO 8601 |
| `total_clicks` | Всего кликов по ссылке |
| `today_clicks` | Кликов за **текущие сутки UTC** (как в админке) |

---

### `POST /api/v1/links/resolve-clicks`

Массовое получение **total_clicks** по URL профилей соцсетей (как в поле «Аккаунт» при создании ссылки). Для интеграции с AccountsStats: до **500** URL за запрос; совпадение label ссылки и переданного URL — после нормализации (регистр, `www`, хвостовой `/`, `@user` в пути TikTok/Threads и т.д.). Если на один аккаунт несколько коротких ссылок с тем же label — клики **суммируются**.

**Тело JSON:**

```json
{
  "profile_urls": [
    "https://www.tiktok.com/@thecapitolverdict",
    "https://www.instagram.com/phil.redpill/"
  ]
}
```

**Ответ `200`:**

```json
{
  "items": [
    { "profile_url": "https://www.tiktok.com/@thecapitolverdict", "total_clicks": 42 },
    { "profile_url": "https://www.instagram.com/phil.redpill/", "total_clicks": 0 }
  ]
}
```

Порядок элементов совпадает с порядком в запросе (после отбрасывания пустых строк).

---

### `POST /api/v1/links`

Создание ссылки.

**Тело JSON:**

```json
{
  "destination_url": "https://example.com/page",
  "label": "https://www.tiktok.com/@user"
}
```

- **`destination_url`** — обязательно; должен начинаться с **`http://`** или **`https://`**.
- **`label`** — необязательно; идентификатор аккаунта (часто URL профиля). Пустая строка → `null`.

**Ответ `201`:** объект ссылки в том же формате, что элемент списка (счётчики кликов для новой ссылки будут нулевыми).

**Ошибка `400`:** неверный URL (не http/https).

---

### `POST /api/v1/links/bulk`

Массовое создание: **одна** целевая ссылка и **несколько аккаунтов** — для каждого аккаунта создаётся отдельная короткая ссылка (как «Массовое создание» в админке).

**Тело JSON** — укажите **`labels`** (массив строк с аккаунтами) **или** **`labels_text`** (многострочная строка, по одному аккаунту на строку):

```json
{
  "destination_url": "https://rumble.com/c/PhilGodlewski",
  "labels": [
    "https://www.tiktok.com/@thecapitolverdict",
    "http://tiktok.com/@sovereigndesk"
  ]
}
```

Альтернатива с текстовым полем:

```json
{
  "destination_url": "https://rumble.com/c/PhilGodlewski",
  "labels_text": "https://www.tiktok.com/@user1\nhttp://tiktok.com/@user2"
}
```

- Пустые строки пропускаются.
- Одинаковые аккаунты (без учёта регистра) объединяются в одну ссылку.
- Не больше **200** аккаунтов за запрос.
- **`destination_url`** — только `http://` или `https://`.

**Ответ `201`:**

```json
{
  "created": 2,
  "items": [ { "...": "объект ссылки как в POST /links" } ]
}
```

**Ошибки `400`:** неверный URL, нет аккаунтов, слишком много аккаунтов, не переданы ни `labels`, ни `labels_text`.

---

### `GET /api/v1/links/{link_id}`

Одна ссылка по UUID. **`404`**, если не найдена.

---

### `PATCH /api/v1/links/{link_id}`

Частичное обновление. В теле можно передать только нужные поля.

**Пример — сменить только аккаунт:**

```json
{ "label": "https://www.tiktok.com/@newuser" }
```

**Сбросить аккаунт:**

```json
{ "label": null }
```

**Сменить целевой URL:**

```json
{ "destination_url": "https://other.example/" }
```

Лишние поля в JSON запрещены (**`422`**, если передать неизвестные ключи).

**Ответ `200`:** актуальный объект ссылки.

---

### `DELETE /api/v1/links/{link_id}`

Удаление ссылки и связанных кликов (каскад в БД). **`204`** без тела. **`404`**, если ссылки нет.

## Статистика по ссылке

### `GET /api/v1/links/{link_id}/stats`

Сводка за период и топ стран.

**Query-параметры:**

| Параметр | Описание |
|----------|----------|
| `preset` | Без дат: **`today`** (по умолчанию), **`week`**, **`all`** (с даты создания ссылки по сегодня, UTC) |
| `from` | Начало периода (если задано вместе с диапазоном — режим «свой период»): дата **`YYYY-MM-DD`** или полная дата-время ISO |
| `to` | Конец периода (для даты **`YYYY-MM-DD`** конец дня считается как начало следующего дня UTC) |

Если указаны **`from`** и/или **`to`** (непустые строки), пресет игнорируется, используется разбор диапазона как в админке.

**Ответ `200`:**

| Поле | Описание |
|------|----------|
| `total` | Число кликов |
| `uniques` | Уникальные посетители (по внутреннему ключу дедупликации) |
| `period_from`, `period_to` | Границы периода для отображения (строки дат) |
| `active_preset` | `today` \| `week` \| `all` \| `custom` |
| `countries` | Массив `{ "code": "DE", "count": 42 }`; пустой код — неизвестная страна |
| `countries_missing_code` | Есть ли в топе записи без кода страны |
| `geoip_db_present` | Как в `/me` |

## Клики по ссылке

### `GET /api/v1/links/{link_id}/clicks`

Постраничный список кликов, сортировка по времени **от новых к старым**.

**Query-параметры:**

| Параметр | По умолчанию | Ограничения |
|----------|--------------|-------------|
| `page` | `1` | ≥ 1 |
| `page_size` | `50` | от 1 до 200 |

**Ответ `200`:** `items`, `total` (всего кликов по ссылке), `page`, `page_size`.

Элемент **`items[]`:** `id`, `link_id`, `created_at`, `ip`, `user_agent`, `referer`, `country_code`, `region`, `city`, `visitor_id`, `dedupe_key`.

---

### `DELETE /api/v1/links/{link_id}/clicks`

Удаляет все клики по ссылке; сама ссылка остаётся. **`204`** без тела.

---

## Экспорт CSV

Ответ — поток **`text/csv; charset=utf-8`** с BOM для Excel. Имя файла передаётся в заголовке **`Content-Disposition`**.

Общие **query-параметры** для обоих экспортов:

| Параметр | Описание |
|----------|----------|
| `link_id` | Необязательный UUID; если указан — только эта ссылка |
| `from`, `to` | Диапазон по времени создания клика (UTC), тот же смысл, что в админском экспорте: при отсутствии значений подставляется окно по умолчанию (конец — «сейчас», начало — 30 суток назад, с поправками при неверном формате дат) |

### `GET /api/v1/export/clicks.csv`

Построчно все клики в диапазоне: колонки  
`id`, `link_id`, `created_at`, `ip`, `user_agent`, `referer`, `country_code`, `region`, `city`, `visitor_id`, `dedupe_key`.

### `GET /api/v1/export/summary.csv`

Агрегация по **`link_id`** и календарному **дню UTC**: колонки `link_id`, `day`, `clicks`, `uniques`.

## Примеры `curl`

Подставьте базовый URL и токен.

```bash
export BASE="https://example.com"
export API_TOKEN="ваш-секрет"

# Список ссылок
curl -sS -H "Authorization: Bearer $API_TOKEN" \
  "$BASE/api/v1/links?limit=20&offset=0"

# Создание одной ссылки
curl -sS -X POST -H "Authorization: Bearer $API_TOKEN" -H "Content-Type: application/json" \
  -d '{"destination_url":"https://example.com","label":"https://www.tiktok.com/@user"}' \
  "$BASE/api/v1/links"

# Массовое создание
curl -sS -X POST -H "Authorization: Bearer $API_TOKEN" -H "Content-Type: application/json" \
  -d '{"destination_url":"https://rumble.com/c/example","labels":["https://www.tiktok.com/@user1","http://tiktok.com/@user2"]}' \
  "$BASE/api/v1/links/bulk"

# Статистика за неделю (пресет)
curl -sS -H "X-Api-Key: $API_TOKEN" \
  "$BASE/api/v1/links/UUID-ССЫЛКИ/stats?preset=week"

# Первая страница кликов
curl -sS -H "Authorization: Bearer $API_TOKEN" \
  "$BASE/api/v1/links/UUID-ССЫЛКИ/clicks?page=1&page_size=25"

# Скачать CSV кликов за период
curl -sS -H "Authorization: Bearer $API_TOKEN" \
  -o clicks.csv \
  "$BASE/api/v1/export/clicks.csv?from=2026-01-01&to=2026-01-31"
```

## Коды ответов (кратко)

| Код | Когда |
|-----|--------|
| `200` / `201` | Успех |
| `204` | Успешное удаление без тела |
| `400` | Неверный URL при создании/обновлении |
| `401` | Нет или неверный токен (при настроенном `API_TOKEN`) |
| `429` | IP заблокирован после неверных токенов **или** превышен лимит запросов в минуту |
| `404` | Сущность не найдена |
| `422` | Ошибка валидации тела или query (FastAPI / Pydantic) |
| `503` | API отключён конфигурацией (`API_TOKEN` не задан) |

---

## Замечания по безопасности

- В **`APP_ENV=production`** обязательны **`API_TOKEN`**, **`SECRET_KEY`** и сменённый **`ADMIN_PASSWORD`**.
- Храните **`API_TOKEN`** только в секретах окружения, не коммитьте в репозиторий.
- Используйте **HTTPS** в продакшене (`SESSION_COOKIE_HTTPS_ONLY=true` для админки).
- Токен даёт полный доступ ко всем операциям API; при компрометации смените его и перезапустите приложение.
- Заголовки **`Authorization: Bearer …`** или **`X-Api-Key`** — не передавайте токен в query-параметрах URL.
